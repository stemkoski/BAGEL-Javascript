Class: Screen

Screen()

A Screen contains the code for each individual game screen, such as a title screen, menu screens, and screens corresponding to each room or level in the game. Each screen stores a Group of entities which will be rendered by the Game class. Classes which extend Screen must implement the initialize and update methods.

Constructor

new Screen()

Creates a Group to store all entities, a variable to reference the Game containing this screen, and calls the initialize method.

Methods

(async) initialize()

Set up all Entity objects and any other initialization code required by this screen.
Must be implemented by extending class.
Note: this method will typically be "async" because "await" should be used when loading textures.

update()

Update all game logic for this screen, such as processing player input, interaction between game entities, and game event conditions (such as win/lose).
Must be implemented by extending class.
